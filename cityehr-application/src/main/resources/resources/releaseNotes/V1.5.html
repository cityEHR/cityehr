V1.5 - New Features

System Level

  1. Use eXist range index for database search
  2. Set up database index on blank start
  3. Option to restrict patient access to patients in user's care team(s)
  4. Details displayed in patient lists are configurable - patient search, cohort search, clinic lists
  5. Enhanced support for triggers on published events (entry/element)
  6. Annotations (record view) can specify effective time
  7. Session control - redirect to home page on error.
  8. Session control - prevent URL injection - redirect to homepage.
  9. Session control - sign on displays message on redirect.

Information Models

  1. Improved support for directory lookup entries
  2. Support calculated enumerated values
  3. Define language for information model

Clinical Data Entry (Forms, Letters, etc)

  1. Record author information when comitting compositions to database
  2. Mouseover shows provenance of compositions in navigation list
  3. Letter templates (MS Word) are configurable by composition type

Patient Search

  1. Search results cleared when criteria are changed
  2. Patients picked from lists must be confirmed before opening patient record
  3. Results can be limited to care team patients only (configurable)
  4. List patients in selected care team
  5. Assign patients to care team

Cohort Search

  1. Improved search performance (up to 100x faster)
  2. Administration search (effective time, author by composition
  3. Set effective time of search (i.e. search on snapshot in time)
  4. Restrict search to current data or all historic data
  5. Rebuild search results when saved searches are loaded
  6. Support saved cohorts (with search criteria removed)
  7. Delete saved searches and cohorts
  8. Print summary of patient cohort lists
  9. Export patient cohorts
  10. Export formats - XML, OWL/XML, spreadsheet (Open Office or Microsoft)
  11. Anonymous patient data export

In-Tray

  1. Click through to launch action (if available)
  2. Click through to patient record
  3. Filters on In-Tray lists
  4. In-tray items can be limited to care team patients only (configurable)

System Administration

  1. Automated regeneration of directories when an information model is updated
  2. Export language packs for information model
  3. Apply language pack to information model
  4. Define common models for an application (shared between specialties)
  5. Check patient lists
  6. Assign listed patients to care team
  7. Redirect norifications
  8. Improvements to user management
  9. Restrict user access to care team patients only
